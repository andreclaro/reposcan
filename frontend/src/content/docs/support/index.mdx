## Getting the Best Support

When contacting support, please include:

### For Scan Issues

1. **Scan ID** - Found in dashboard URL
2. **Repository URL** - e.g., `github.com/org/repo`
3. **Error messages** - Copy exact text
4. **What you expected** vs **what happened**

### For API Issues

1. **Request ID** - In error response
2. **Endpoint** - e.g., `POST /scan`
3. **Timestamp** - When error occurred
4. **Request/response** (remove sensitive data)

### For Integration Issues

1. **Integration type** - GitHub, CI/CD, API
2. **Configuration** - Relevant settings
3. **Logs** - Error messages
4. **Steps to reproduce**

## Security Issues

### Reporting Vulnerabilities

If you discover a security issue:

1. **DO NOT** open a public issue
2. Email: security@securitykit.com
3. Include detailed description
4. Allow 48h for initial response
