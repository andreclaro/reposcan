# AI-Powered Analysis

SecurityKit uses AI to analyze scan results and provide actionable insights beyond raw findings. Get prioritized recommendations, risk scores, and executive summaries.

## How It Works

After a scan completes, our AI analyzes:
- All findings and their severity
- Code context around vulnerabilities
- Historical patterns
- Industry best practices

The result is an intelligent assessment that helps you focus on what matters most.

## Features

### Risk Scoring
Overall risk score from 0-100 based on:
- Severity distribution
- Exploitability
- Business impact
- Attack surface exposure

### Executive Summary
High-level overview including:
- Total findings by severity
- Most critical issues
- Trends vs. previous scans
- Recommended actions

### Prioritized Recommendations
Issues ranked by:
- Security impact
- Fix complexity
- Business risk
- Exploitation likelihood

### Fix Suggestions
For each finding:
- Explanation of the vulnerability
- Code-level fix examples
- Effort estimation
- Related resources

## Enabling AI Analysis

### Requirements
- AI analysis enabled in your plan
- Valid API key (Anthropic, OpenAI, or Kimi)
- Completed scan with findings

### Configuration

```bash
# Environment variables
AI_ANALYSIS_ENABLED=true
AI_PROVIDER=anthropic
AI_MODEL=claude-3-sonnet-20240229
ANTHROPIC_API_KEY=sk-ant-api03-...
```

### Generate Analysis

1. Complete a scan
2. Navigate to the **AI Analysis** tab
3. Click **"Generate Analysis"**
4. Wait 1-2 minutes for processing

## Privacy & Security

- Source code is processed securely
- No data retained by AI providers
- Analysis stays within your account
- Compliant with SOC 2 requirements

## Limitations

- Requires scan to complete first
- Analysis time: 1-3 minutes
- Available for scans with findings
- Subject to API rate limits

## Interpreting Results

### Risk Score Guidelines

| Score | Level | Meaning |
|-------|-------|---------|
| 0-25 | Low | Minor issues, generally safe |
| 26-50 | Medium | Some concerns, address in next sprint |
| 51-75 | High | Significant risks, prioritize fixes |
| 76-100 | Critical | Immediate action required |

### Fix Priority

Issues are prioritized by:
1. **Exploitability** - How easy to exploit
2. **Impact** - Potential damage
3. **Exposure** - Public vs. internal
4. **Complexity** - Difficulty to fix

## Best Practices

1. **Generate for every scan** - Track trends over time
2. **Review top priorities** - Focus on high-impact fixes
3. **Share summaries** - Communicate with stakeholders
4. **Track improvements** - Monitor risk score changes
5. **Combine with manual review** - AI augments, not replaces

## Troubleshooting

### Analysis Not Available
- Ensure AI is enabled in settings
- Check API key is valid
- Scan must complete successfully

### Slow Generation
- Large codebases take longer
- Consider scanning smaller subsets
- Check API rate limits
