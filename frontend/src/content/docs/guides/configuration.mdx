# Configuration

Configure SecurityKit using environment variables.

## Required Settings

### Database

```bash
DATABASE_URL=postgresql://user:password@host:port/database
```

**Example:**
```bash
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/sec_audit
```

### Task Queue

```bash
REDIS_URL=redis://host:port/database
```

**Example:**
```bash
REDIS_URL=redis://localhost:6379/0
```

### Results Storage

```bash
RESULTS_DIR=./results
```

## Optional Settings

### AI Analysis

Enable AI-powered security analysis:

```bash
AI_ANALYSIS_ENABLED=true
AI_PROVIDER=anthropic          # anthropic, openai, or kimi
AI_MODEL=claude-3-sonnet-20240229
ANTHROPIC_API_KEY=sk-ant-api03-...
```

**Available Models:**

| Provider | Model | Description |
|----------|-------|-------------|
| Anthropic | claude-3-opus-20240229 | Most capable |
| Anthropic | claude-3-sonnet-20240229 | Balanced (default) |
| Anthropic | claude-3-haiku-20240307 | Fastest |
| OpenAI | gpt-4-turbo-preview | High capability |
| OpenAI | gpt-3.5-turbo | Cost effective |
| Kimi | kimi-k2.5 | Large context |

### Storage Backend

```bash
STORAGE_BACKEND=local          # local, s3, or none
STORAGE_BASE_PATH=./results    # For local storage
```

**S3 Configuration:**

```bash
STORAGE_BACKEND=s3
S3_BUCKET=my-bucket
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=...
S3_ENDPOINT_URL=https://s3.amazonaws.com  # Optional (for MinIO, etc.)
```

### GitHub Integration

```bash
GITHUB_CLIENT_ID=your_client_id
GITHUB_CLIENT_SECRET=your_client_secret
NEXTAUTH_SECRET=$(openssl rand -base64 32)
GITHUB_TOKEN=ghp_...  # Optional, for API access
```

**Generate NEXTAUTH_SECRET:**

```bash
openssl rand -base64 32
```

### Next.js Settings

```bash
FASTAPI_BASE_URL=http://localhost:8000
NEXT_PUBLIC_APP_URL=http://localhost:3003
```

### Contact Form

```bash
RESEND_API_KEY=re_...
RESEND_FROM="SecurityKit <support@your-domain.com>"
CONTACT_EMAIL=support@your-domain.com
```

### Beta Mode

Require admin approval for new accounts:

```bash
BETA_MODE_ENABLED=true
ADMIN_EMAIL=admin@example.com,admin2@example.com
```

## Scan Timeouts

Configure timeouts for scan operations:

```bash
SEC_AUDIT_CLONE_TIMEOUT=600          # Git clone (seconds)
SEC_AUDIT_SUBMODULES_TIMEOUT=300     # Submodule update
SEC_AUDIT_SEMGREP_TIMEOUT=600        # SAST scan
SEC_AUDIT_TRIVY_TIMEOUT=300          # Container scan
SEC_AUDIT_DOCKER_BUILD_TIMEOUT=600   # Docker build
SEC_AUDIT_ECOSYSTEM_AUDIT_TIMEOUT=300 # Dependency audit
```

## Security Best Practices

### 1. Use .env.local

Add `.env.local` to `.gitignore`:

```bash
# .gitignore
.env.local
.env
*.key
```

### 2. Rotate Secrets Regularly

- API keys: Every 90 days
- Database passwords: Every 180 days
- OAuth secrets: After team changes

### 3. Use Environment-Specific Configs

```bash
# Development
.env.development.local

# Staging
.env.staging.local

# Production
# Use secrets manager (AWS Secrets Manager, Vault, etc.)
```

### 4. Limit API Key Permissions

Create separate keys for:
- CI/CD pipelines (scans:write only)
- Dashboard access (read-only)
- Admin operations (full access)

### 5. Monitor Usage

Track API key usage:
- Unusual patterns
- Failed authentication attempts
- Rate limit hits

## Environment Variable Priority

Variables are loaded in this order (later overrides earlier):

1. `.env` (default)
2. `.env.local` (gitignored, local overrides)
3. `.env.{environment}` (development, production, etc.)
4. `.env.{environment}.local` (environment + local)
5. System environment variables (highest priority)

## Validation

### Check Configuration

```bash
# Python
python -c "from backend.src.audit.utils import validate_config; validate_config()"

# Or via API
curl http://localhost:8000/health
```

### Required Variables Check

```bash
#!/bin/bash
required=("DATABASE_URL" "REDIS_URL" "NEXTAUTH_SECRET")

for var in "${required[@]}"; do
  if [ -z "${!var}" ]; then
    echo "Error: $var is not set"
    exit 1
  fi
done

echo "All required variables are set"
```

## Troubleshooting

### Variable Not Found

```bash
# Check if set
echo $DATABASE_URL

# Check loaded config
docker compose exec api env | grep DATABASE
```

### Connection Errors

```bash
# Test database
psql $DATABASE_URL -c "SELECT 1"

# Test Redis
redis-cli -u $REDIS_URL ping
```
