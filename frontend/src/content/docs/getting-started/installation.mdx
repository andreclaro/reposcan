# Installation

You can use SecurityKit as a hosted service or run it in your own infrastructure.

## Option 1: Cloud (Recommended)

The easiest way to get started. No installation required.

1. Sign up at [SecurityKit Cloud](https://your-domain.com)
2. Connect your GitHub account
3. Start scanning immediately

## Option 2: Docker Compose

Run the complete SecurityKit stack locally using Docker Compose.

### Prerequisites

- Docker Engine 20.10+
- Docker Compose 2.0+
- 4GB RAM minimum

### Quick Start

```bash
# Clone the repository
git clone https://github.com/your-org/sec-audit-repos.git
cd sec-audit-repos

# Create environment file
cp frontend/.env.local.example frontend/.env.local

# Edit with your values
# Required: DATABASE_URL, REDIS_URL

# Start all services
docker compose -f docker/docker-compose.yml up -d
```

### Services

| Service | Port | Description |
|---------|------|-------------|
| API | 8000 | FastAPI backend |
| Worker | - | Celery task processor |
| PostgreSQL | 5432 | Database |
| Redis | 6379 | Task queue |

### Verify Installation

```bash
# Check health
curl http://localhost:8000/health

# View logs
docker compose -f docker/docker-compose.yml logs -f
```

## Option 3: Manual Installation

### Backend (Python)

```bash
# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
cd backend
pip install -r requirements.txt

# Set environment variables
export DATABASE_URL="postgresql://postgres:postgres@localhost:5432/sec_audit"
export REDIS_URL="redis://localhost:6379/0"
export RESULTS_DIR="./results"
```

### Frontend (Next.js)

```bash
cd frontend
pnpm install

# Create local environment
cp .env.local.example .env.local
# Edit with your API URL and auth settings
```

### Required System Tools

Install these security scanners:

```bash
# macOS
brew install semgrep trivy tfsec checkov tflint

# Ubuntu/Debian
# Install semgrep
pip install semgrep

# Install Trivy
curl -sfL https://raw.githubusercontent.com/aquasecurity/trivy/main/contrib/install.sh | sh

# Install tfsec, checkov, tflint
# See respective documentation for latest instructions
```

## Configuration

### Minimum Configuration

```bash
# Database
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/sec_audit

# Task Queue
REDIS_URL=redis://localhost:6379/0

# Results Storage
RESULTS_DIR=./results
```

```bash

GITHUB_CLIENT_SECRET=your_client_secret
NEXTAUTH_SECRET=$(openssl rand -base64 32)
```

## Running Locally

### Start Services

```bash
# Terminal 1: Start API
cd backend
uvicorn api.main:app --reload --port 8000

# Terminal 2: Start Worker
cd backend
celery -A tasks.scan_worker worker --loglevel=info

# Terminal 3: Start Frontend
cd frontend
pnpm dev
```

### Access the Application

- **Web Dashboard**: http://localhost:3003
- **API Docs**: http://localhost:8000/docs
- **API Base**: http://localhost:8000

## Troubleshooting

### Port Conflicts

If ports are already in use, modify `docker-compose.yml`:

```yaml
services:
  api:
    ports:
      - "8001:8000"  # Change 8001 to available port
```

### Permission Issues

```bash
# Fix Docker socket permissions (Linux)
sudo usermod -aG docker $USER
# Log out and back in
```

### Database Connection Failed

```bash
# Check PostgreSQL is running
docker compose ps

# View logs
docker compose logs postgres
```
